Данный проект показывает работу с развлекательным приложением.

Команда разработчиков рекомендательных систем создала новый алгоритм рекомендаций, который, по их мнению, будет показывать более интересный контент для каждого пользователя. Требуется рассчитать параметры A/B-теста, который позволит проверить эту гипотезу, и проанализировать его результаты.

В ходе проделанной работы были выполнены следующие шаги:
1 ЭТАП. Знакомство с данными:
1. Импорт необходимых библиотек, загрузка файлов. 
2. Анализ изменения кол-ва уникальных и зарегистрированных пользователей по дням за наблюдаемый период: Построен график динамики пользователей по дням.
3. Анализ числа просмотренных страниц: Построена столбчатая диаграмма сравнения числа страниц. 
4. Построен график доли успешных сессий (более 4 страниц за сеанс) по дням, сформулированы первичные выводы. 

2 ЭТАП. Проведение А/В теста:
1. Сформулированы нулевая и альтернативная гипотезы.
2. Произведен расчет размера выборки для каждой группы для проведения А/В теста через solve_power()
3. Произведен расчет длительности А/В теста.
4. Проведена проверка однородности распределения количества пользователей по каждой из групп.
5. Проведена проверка на независимость каждой из групп.
6. Проведена проверка на равномерность распределения пользователей по устройствам, регионам в каждой из групп, сформулированы выводы. 
7. Проведена проверка корректности результатов теста через t-тест Стьюдента, сформулированы выводы о разнице долей успешных сессий в каждой из групп.
8. Проведена проверка на статистическую значимость изменения ключевой метрики после проведения теста с помощью Z-теста пропорций.  
9. Сформулированы финальные выводы.
